# vibe-coder-server 컨테이너 글로벌 규칙

이 파일은 컨테이너 안의 `/home/vibe/.claude/CLAUDE.md` 로 마운트되어, 컨테이너에서 실행되는
Claude Code 및 Codex 등 AI 코딩 세션에 글로벌로 적용된다. Codex 는
`/home/vibe/.codex/AGENTS.md` symlink 를 통해 같은 내용을 읽는다.

> 이 파일은 서버 최초 기동 시 자동 시드되는 기본 템플릿이다. 한 번 존재하면 서버가 다시
> 덮어쓰지 않는다. 플랫폼별 빌드, UI, 배포, 도구 규칙은 프로젝트 생성 시 선택한 platform
> engine 이 프로젝트 루트의 지역 `CLAUDE.md`에 주입한다.

## 1. 응답 언어

- 사용자 응답은 항상 한국어 존댓말로 작성한다.
- 코드, 명령어, 식별자, 고유명사는 원문을 유지한다.

## 2. 작업 방식

- 코드 변경 후 가능한 범위에서 컴파일, 빌드, 관련 테스트를 실행한다.
- 실행하지 못한 검증은 이유와 함께 명시한다.
- 변경 파일, 핵심 결정, 남은 리스크를 짧게 요약한다.
- 인터랙티브 입력을 기다리는 명령, 무기한 실행 명령, watch/REPL/TUI 명령을 피한다.
- 확인이 필요하면 응답 끝에 선택지와 권장안을 남기고 멈춘다. 실행 중 stdin 대기는 금지한다.

## 3. 전역/지역 지침 경계

- 전역 지침은 모든 모바일 프로젝트에 공통인 운영, 보안, 문서화, 검증 원칙만 담는다.
- 특정 플랫폼의 빌드 명령, UI 프레임워크, 패키지 구조, signing, store upload, 런타임 기기
  규칙은 지역 `CLAUDE.md`만 따른다.
- 프로젝트 타입을 추측하지 않는다. 프로젝트 루트의 지역 `CLAUDE.md`, 설정 파일, 서버가 제공한
  project metadata를 먼저 확인한다.
- 다른 플랫폼의 도구나 prompt를 자동 호출하지 않는다.

## 4. 보안

- secret, token, signing asset, API key, password, private key를 로그, 문서, 테스트 fixture,
  git commit에 남기지 않는다.
- workspace boundary 밖 파일을 임의로 읽거나 쓰지 않는다.
- raw shell 문자열을 사용자 입력으로 직접 구성하지 않는다.
- 외부 쓰기 권한이 있는 MCP, release, publish, deploy 작업은 사용자가 명시적으로 요청한 경우에만
  수행한다.

## 5. 도구와 MCP

- 최신 라이브러리, SDK, CLI, cloud/API 동작이 필요한 경우 공식 문서 또는 연결된 문서 MCP를 먼저
  확인한다.
- 전역 기본 MCP는 플랫폼 중립 도구만 사용한다.
- 플랫폼별 MCP, skill, agent 추천은 프로젝트의 platform engine/tooling profile을 따른다.
- 조건부 도구는 preflight, secret 등록, 사용자의 명시 요청이 충족된 경우에만 사용한다.

## 6. 모바일 공통 품질 기준

- 접근성 label, 터치 타깃, 작은 화면, 긴 텍스트, 로딩, 빈 상태, 오류 상태, 권한 거부 상태를
  작업 범위에 맞게 확인한다.
- 불필요한 의존성, 과한 추상화, 레거시 코드, 죽은 코드, 중복 도구 설치를 피한다.
- 기존 프로젝트의 구조와 디자인 언어를 먼저 확인하고 그 안에서 개선한다.

## 7. Git과 문서

- 변경이 사용자 가시 동작, 운영 정책, API, wire shape, 배포 방식, 프로젝트 생성 방식에 영향을 주면
  관련 문서를 함께 갱신한다.
- 레거시 설명과 맞지 않는 예시는 발견 즉시 정리한다.
- 커밋/푸시는 사용자가 명시적으로 요청한 경우에만 수행한다.
